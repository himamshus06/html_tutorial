CSS - Cascading Style Sheets


Yesterday, you saw an example to see the importance of css. without css the website is just bland and sad. Css is basically stylesheets that give style and beauty to sad html elements. Cascading refers to how css properties are applied based on their priorities. More on that later.

There are 3 ways to use css. (Why three three ways? it is useful actually. Helps with overwriting unwanted styles for individual elements)

  1. External CSS: Create a css file (outside html document) and save it with '.css' extension. Like: "filename.css". Then link it to ur html document in the head section (how? we'll see later). btw here the stylesheet is named style.css and its linked using the <link> tag


  2. Internal CSS: use the <style> css stuff </style> tag in the head section. In this image you can see the style tag and css being defined within style tag. The style tag is in the head section.



  3. Inline CSS: use the style as attribute in html elements like:
    <p atrribute_name = "attribute value"> text </p>

    or, in this case,

    <img style = "css stuff">

    btw img tag is used to put images into the webpage. Later we'll see about it




In external and internal css, we use "selectors" to select a particular element that we wanna edit and beautify. Selectors can be either elements themselves, like h1, p, div, that select all the content placed within any number of these selected tags, OR class names OR ID names. The syntax (format) to use css is, (in external css)

selector{
properties;
}

element selector


here im applying certain style to the "p" elements by using element selector 'p'.

class selector


In html and css, class is basically just a name given to the elements whose css we want to define separately without affecting other contents. class names can be given to just about any html element. p tag, h1 tag, div tag, you name it. Class name is given as an attribute. An attribute is any property that is put between the name of the element and the > bracket like so:

consider a p tag;

<p atrribute_name = "attribute value"> text </p>

or, in this case,

<p class = "classname"> text </p>

example:

this para is given background-color of "aqua" using class selector. The code for that is:

.classname{
background-color:aqua;
}



ID selector


In html and css, ID is basically just a name given to the elements whose css we want to define separately without affecting other contents. How is it different than class ?

  1. The same class names can be used for many elements. For example, i can have a paragraph, a heading and a table all with the same class name and define common styles for all of them. One ID name can be used only once. It makes the name more unique

  2. ID names are needed when u wanna use javascript or use links for a particular section in the same webpage
<p class = "classname"> text </p>



this para is given background-color of "blue" and text-color "white" using id selector. The code for that is:

#idname{
background-color:beige;
}





NOTE:



note that for class name selector we use '.' (dot before classname):

.classname{
background-color:aqua;
}

and for idname we use '#' (hash or pound sign before idname):

#idname{
background-color:beige;
}
